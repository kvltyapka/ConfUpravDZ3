# Config Language Parser

Проект представляет собой парсер для конфигурационных файлов, который преобразует YAML-файлы в специфичный формат конфигурации с использованием определённого синтаксиса.

## Описание

Этот парсер обрабатывает YAML файлы с конфигурацией, преобразуя их в формат, аналогичный следующему:

### Пример входных данных (YAML):

```yaml
name: "Web Server Config"
version: 2.1
features:
  - logging
  - caching
  - compression
settings:
  host: "localhost"
  port: 8080
  ssl_enabled: true
max_connections: 100
expression: "?[max_connections + 10]"
```

Пример выходных данных (Конфигурация):

```text
name = "Web Server Config";
version = 2.1;
features = ("logging", "caching", "compression");
settings{
    host = "localhost";
    port = 8080;
    ssl_enabled = true;
};
max_connections = 100;
expression = 110;
Парсер поддерживает базовые типы данных: строки, числа, булевы значения, списки и словари. Также поддерживается выполнение выражений, как например, ?[max_connections + 10].
```

### Запустите парсер:

Для преобразования вашего YAML файла в нужный конфигурационный формат используйте команду:

```bash
cat <path_to_yaml_file> | python main.py
```

### Тестирование

Для тестирования парсера используется библиотека unittest.
Запуск тестов
Чтобы запустить тесты, выполните команду:

```bash
python -m unittest test_config_language.py
```

Тесты будут проверять корректность преобразования различных конфигурационных файлов в нужный формат. Для каждого теста будет выведено, что ожидалось и что было получено, если результаты не совпадают.
